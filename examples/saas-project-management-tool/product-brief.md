# Tessera — fictional product brief

> ⚠️ This is a **fictional product** used as Personakit's reference example.
> No real company, customer, or competitor is described here.

## Product

**Tessera** is a mid-market project management SaaS that competes on:

1. **Native integrations** with the dev/design/sales tools mid-market teams
   already use (Linear, Figma, HubSpot, Slack, Notion).
2. **AI-assisted planning** — auto-Gantt generation from existing task data,
   capacity-aware scheduling, and weekly "what slipped, why" digests.
3. **Mid-market pricing** — explicitly NOT priced for the enterprise-ELA
   motion. Self-serve up to 200 seats with one-click annual upgrade.

## Target segment

- Companies with 50-500 employees
- B2B SaaS, agencies, and product-led mid-market software shops
- Teams that have outgrown Trello/Asana but explicitly do NOT want Jira

## The single biggest user problem

> "Our PMs spend more time keeping the project tool in sync than running
> the project."

Existing tools force users to re-enter the same status across calendar,
spec doc, ticket tracker, and slide deck. Tessera's wedge is that one
status update propagates everywhere.

## Pricing today

- **Starter** — $0 up to 5 users
- **Team** — $12/user/month, billed annually
- **Business** — $24/user/month with SSO and audit log

## What we want Personakit to validate

- Whether **auto-Gantt generation from existing task data** is a paid add-on,
  a Business-tier inclusion, or table stakes
- Whether the segment will pay for **AI-assisted weekly digests**
- Where the integration story breaks down by company size

## Known competitors (named for prompt grounding only — no proprietary data)
